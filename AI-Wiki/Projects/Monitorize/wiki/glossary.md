# Monitorize Glossary

## ADB

Android Debug Bridge. Monitorize uses ADB reverse mappings for USB streaming.

## Extend Mode

Mode where Monitorize creates or requests an additional virtual/headless display and streams it to the client.

## GStreamer

Media pipeline used by the Linux app to encode the PipeWire capture as low-latency H.264.

## KDE Virtual Display

A virtual output created through KDE Plasma 6.7+ ScreenCast portal support. Monitorize detects and configures the safe new `Virtual-*` output.

## Mirror Mode

Mode where Monitorize captures an existing display instead of creating a new virtual display.

## PipeWire

Linux media infrastructure used for display capture through portals and desktop-specific streamers.

## TLS Pairing

Encrypted Wi-Fi flow where Android pairs with the Linux app using a PIN, pinned certificate fingerprint, and authentication token.

## uinput

Linux input subsystem interface used by Monitorize to inject touch, stylus, or mouse input into the desktop session.

## Zeroconf

Local network service discovery used for advertising Monitorize Wi-Fi stream availability.
