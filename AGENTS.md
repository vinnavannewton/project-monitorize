# Codex Project Instructions

This repository has a Codex-maintained Obsidian wiki at `AI-Wiki/Projects/Monitorize/wiki`.

## Wiki Workflow

- When learning durable project knowledge, update the relevant wiki page.
- Read `AI-Wiki/Projects/Monitorize/wiki/index.md` first when using the wiki for project context.
- Keep wiki pages short, factual, and useful for future Codex sessions.
- Use Obsidian links such as `[[architecture]]`, `[[decisions]]`, and `[[open-questions]]`.
- Update `AI-Wiki/Projects/Monitorize/wiki/index.md` when pages are added or meaningfully changed.
- Append `AI-Wiki/Projects/Monitorize/wiki/log.md` after meaningful discoveries, ingests, architecture notes, decisions, or handoff updates.
- Treat `AI-Wiki/Projects/Monitorize/raw` as immutable source material. Read from it, but do not rewrite original sources unless the user explicitly asks.

## Project Notes

- Monitorize is a Linux and Android project for using an Android device, laptop, or PC as a secondary monitor for a Linux desktop.
- Prefer existing repository docs as sources before inventing wiki content.
- Keep implementation changes separate from wiki maintenance unless the user asks for both.
